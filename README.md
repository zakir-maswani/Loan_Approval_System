## Overview

This project builds an end-to-end loan approval prediction pipeline — from raw data ingestion and exploratory analysis through model training, evaluation, and a live prediction interface. Given a set of applicant attributes (income, credit score, employment status, etc.), the model predicts whether a loan application will be **Approved** or **Not Approved**.

## Demo

Launch the Streamlit app locally (see [Usage](#usage)) and interact with the sidebar to enter applicant details. Click **Predict** to receive an instant approval decision.

## Dataset

The dataset (`loan_approval_data.csv`) contains applicant records with the following characteristics:

| Property | Detail |
|---|---|
| Source | Local CSV file |
| Target column | `Loan_Approved` (Yes / No) |
| Dropped column | `Applicant_ID` (identifier, not predictive) |

Missing values are handled prior to modelling:
- **Categorical columns** — imputed with the **mode** (most frequent value)
- **Numeric columns** — imputed with the **median**

No duplicate records were found in the dataset.

## Exploratory Data Analysis

The notebook performs thorough univariate and bivariate analysis. Key findings include:

**Gender**
- Male applicants (~62%) substantially outnumber female applicants (~38%).
- Males hold 58% of approved loans; females 42%.

**Marital Status**
- Married applicants account for 62% of approvals and 65% of rejections, reflecting their dominance in the overall dataset.

**Employment Status**
- Salaried applicants receive ~48% of all approved loans — nearly twice the next group (Contract, ~27%).
- Unemployed applicants have the lowest approval rate.

**Credit Score & DTI Ratio**
- Strongest predictors of approval. Approved applicants have a notably higher median credit score.
- DTI Ratio shows a strong negative correlation with approval (-0.44).

**Income**
- Both applicant and co-applicant income span wide ranges for approved and rejected loans, confirming that income alone does not determine approval.

**Correlation highlights**
- `Credit_Score`: +0.45
- `DTI_Ratio`: -0.44
- `Loan_Amount`: +0.13
- `Applicant_Income`: +0.12

## Model

| Parameter | Value |
|---|---|
| Algorithm | `RandomForestClassifier` |
| `n_estimators` | 400 |
| `min_samples_split` | 5 |
| `random_state` | 42 |
| Train / Test split | 70% / 30% (stratified) |

Categorical features are ordinally encoded before training using the following mappings:

| Feature | Encoding |
|---|---|
| Gender | Male → 1, Female → 0 |
| Marital Status | Married → 1, Single → 0 |
| Employment Status | Salaried → 1, Self-employed → 2, Contract → 3, Unemployed → 0 |
| Loan Purpose | Personal → 1, Car → 2, Business → 3, Home → 4, Education → 5 |
| Property Area | Rural → 0, Urban → 1, Semiurban → 2 |
| Education Level | Not Graduate → 0, Graduate → 1 |
| Employer Category | Unemployed → 0, Business → 1, Private → 2, MNC → 3, Government → 4 |

The trained model is serialized to `model.pkl` using **joblib**.

## Installation

### Prerequisites

- Python 3.8+
- pip

### Steps

1. **Clone the repository**

```bash
git clone https://github.com/your-username/loan-approval-prediction.git
cd loan-approval-prediction
```

2. **Create and activate a virtual environment** *(recommended)*

```bash
python -m venv venv
# macOS / Linux
source venv/bin/activate
# Windows
venv\Scripts\activate
```

3. **Install dependencies**

```bash
pip install -r requirements.txt
```

> **Minimum required packages:**
> `pandas`, `scikit-learn`, `matplotlib`, `seaborn`, `joblib`, `streamlit`

## Usage

### Run the Streamlit App

```bash
streamlit run app.py
```

Then open [http://localhost:8501](http://localhost:8501) in your browser.

### Re-train the Model

Open and run the notebook end-to-end:

```bash
jupyter notebook loan_approval_system.ipynb
```

This will regenerate `model.pkl` from the dataset.
